# 멱등성
`멱등성`이란 동일한 요청을 여러 번 보내도 결과가 같은 것을 의미합니다.
# HTTP Method 멱등성
`HTTP Method`에서의 멱등성은 동일한 요청을 여러 번 보내도 요청을 보낸 `서버의 상태가 동일`한 것을 의미합니다. 
`HTTP Method`는 `POST`, `GET`, `DELETE`, `PUT`, `PATCH` 등이 있는데요, 이 중에서 `POST`와 `PATCH`는 멱등성이 없고, 그 외에 `GET`, `DELETE`, `PUT`은 멱등성이 있습니다.
# POST
`POST Method`는 요청을 보낼 시 서버가 요청을 받을 때마다 새로운 자원을 생성하기 때문에 동일한 요청을 여러 번 보내게 되면 매번 서버의 상태가 바뀌게 됩니다. 그러므로 `멱등성`이 없다고 할 수 있습니다.
# GET
`GET Method`는 서버에 요청을 보내어 원하는 자원을 조회할 때 사용합니다. 
이 때 단순히 서버에서 자원을 조회하고 쓰기 작업은 이루어지지 않기 때문에 서버에 동일한 요청을 보내어도 똑같은 자원이 조회되어 `멱등성`을 가지게 됩니다.
# DELETE
`DELETE Method`는 서버에 요청을 보내어 삭제하고 싶은 자원을 삭제할 때 사용하는데요, 
이 때 동일한 삭제 요청을 여러 번 보내게 되면 첫 요청에서 이미 자원이 삭제되었기 때문에 또 다시 요청을 보내어도 서버의 상태가 변하지 않아 `멱등성`이 있다고 말할 수 있습니다.
# PUT
`PUT Method`는 자원을 전체 수정할 때 사용합니다. 
`HTTP` 요청을 보낼 때 Body로 수정하려는 자원의 `모든 정보`를 요청하게 되면 요청한 정보로 자원이 덮어쓰이게 됩니다. 이 때 동일한 수정 요청을 여러 번 보내더라도 해당 자원은 항상 동일한 정보로 덮어씌워지기 때문에 서버의 상태가 변하지 않아 `멱등성`이 있다고 말할 수 있습니다.
# PATCH
`PATCH Method`는 자원을 일부 수정할 떄 사용합니다. 
이 `PATCH Method`는 구현 방식에 따라 멱등성이 갈립니다. 만약 요청을 보낼 때 해당 자원에 대해서 점수가 1씩 증가하는 것과 같은 추가적인 기능이 수행된다면 동일한 요청을 보내어도 서버의 상태가 달라져 멱등성이 없다고 판단되지만, 추가적인 기능 없이 수정하려는 자원의 정보에 대해서만 동일한 요청을 여러 번 보내게 된다면 서버의 상태가 유지되어 `멱등성`을 가지게 됩니다.
