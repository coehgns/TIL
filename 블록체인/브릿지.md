# 브릿지
- 브릿지는 서로 다른 블록 체인 네트워크에서 자산과 데이터를 연결할 수 있는 다리 역할을 합니다.
- 브릿지를 사용하면 서로 다른 블록 체인 네트워크에서 자산을 더 빠르고 효율적으로 전송할 수 있습니다.

블록 체인 시장이 발전함에 따라 여러 블록 체인 프로토콜이 나오고 있습니다. 이로 인해 블록 체인 네트워크 간의 자산 이동에 대한 수요도 늘어나고 있습니다.

브릿지가 나오기 전에는 각 블록 체인 네트워크는 고유의 언어와 규칙, 프로토콜 등을 사용하다보니 상호 작용하는데에 있어어려움이 있었습니다. 하지만 브릿지를 사용하면 `BTC`를 브릿지를 통해 이더리움 네트워크로 가져와 `ETH`로 교환할 수 있고, `CEX`와 같은` 중앙화 거래소`에서 인출하고 입급하는 것보다 더 저렴하고 빠르게 처리 가능합니다.
# 브릿지 종류
브릿지는 `중앙화된 브릿지`(`Trusted Bridges`)와 `탈중앙화된 브릿지`(`Decentralized Bridges`)로 유형이 나뉩니다.
## 중앙화된 브릿지
- 중앙화된 브릿지는 전환하려는 자산을 중개자에게 맡기고, 중개자가 해당 자산을 관리하는 방식으로 이루어집니다.
- 중개자는 특정한 기관이나 플랫폼이 맡게 되고, 중개자가 거래를 검증하고 관리하게 됩니다.
	- `바이낸스 브릿지`, `폴리곤 POS 브릿지`, `WBTC 브릿지` 등이 대표적인 예입니다.

예를 들어 `BTC`를 이더리움 네트워크로 가져오려고 할 때, `BTC`는 중앙화된 브릿지에 잠금이 걸리게 되고, 중앙화된 브릿지는 사용자에게 `WBTC라`는 래핑된 `BTC`가 이더리움 네트워크에 발행됩니다. 이러한 `WBTC`는 비트코인과 동일한 가치가 되고, `DApp`이나 `DeFi` 프로토콜에서 사용할 수 있습니다. 

이후 사용자가 다시 원래 체인으로 돌아가고 싶은 경우 이더리움 네트워크에서 `WBTC`를 소각하고 잠겨있던 `BTC`를 돌려 받게 됩니다.
## 탈중앙화된 브릿지
- 탈중앙화된 브릿지는 중앙화된 중개자 없이 `스마트 계약`을 통해 자동 검증 후 거래가 이루어집니다.
- 중개자 없기에 스마트 계약으로 인해 사용자가 자산 제어권을 계속해서 가지게 되지만, 프로그래밍 오류나 버그가 발생할 수 있고, 해커에게 악용될 수 있습니다.

예를 들어 `BTC`를 이더리움 네트워크로 가져오려고 할 때, 스마트 계약으로 인해서 자동으로 탈중앙화된 브릿지에 BTC에 잠금을 걸고, `WBTC`를 발행하게 됩니다.
