# 이상 현상(Anomaly)
- `이상 현상`은 데이터베이스에서 `불필요한 데이터 중복`으로 인해서 데이터를 삽입, 삭제, 수정할 때 발생하는 오류를 말합니다.
- 데이터베이스 정규화를 거치지 않았을 때 발생하기 때문에 이를 해결하기 위해 데이터베이스 `정규화`를 거쳐야 합니다.
- 이상 현상의 종류에는 `삽입 이상`, `갱신 이상`, `삭제 이상`이 있습니다.
## 삽입 이상
- 데이터베이스에 데이터를 삽입 할 때 의도와 다른 값들도 `삽입`되는 현상입니다.
유저 이름과 유저 전화번호, 동아리 정보가 저장되고, `기본키`가 `유저 Id`와 `동아리 Id`인 테이블이 있다고 가정할 때, 유저의 정보는 저장하고 싶지만 유저가 동아리를 소속하지 않았거나, 유저의 동아리 정보를 모를 때 삽입할 수 없게 됩니다.
## 갱신 이상
- 데이터베이스에서 데이터를 수정할 때 많은 튜플의 데이터를 수정해야하는 현상입니다.
갱신 이상이 있는 경우, 유저 이름과 유저의 지원서 폼이 저장되어있는 테이블이 있다고 가정할 때, 유저가 개명하게 된다면 해당 유저의 모든 튜플에서 유저 이름을 수정해주어야 하게 됩니다.
## 삭제 이상
- 데이터베이스에서 데이터 삭제 할 때 의도와 다른 데이터들도 함께 `삭제`되는 현상을 말합니다.
유저 이름, 동아리 공고, 전공을 저장하는 테이블이 있다고 가정할 때, 사용자가 동아리 공고 내리기 위해서 삭제할 경우, 사용자의 의도와 다르게 유저 이름과 전공도 함께 삭제되게 됩니다.
